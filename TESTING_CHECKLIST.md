# Blogify Backend - Testing Checklist\n\n## Pre-Deployment Testing\n\n### 1. Environment Setup\n- [ ] PostgreSQL is installed and running\n- [ ] `.env` file created with all required variables\n- [ ] Database URL is correct and accessible\n- [ ] Google OAuth credentials obtained from Cloud Console\n- [ ] JWT_SECRET is set to a strong random string\n\n### 2. Dependencies\n- [ ] Run `npm install` successfully\n- [ ] No conflicting dependency versions\n- [ ] Arctic library installed (for Google OAuth)\n- [ ] All required packages present in package.json\n\n### 3. Server Startup\n- [ ] Server starts without errors: `npm start`\n- [ ] All tables created automatically\n- [ ] Console shows green checkmarks (✓) for table creation\n- [ ] Server listening on correct port (8000 by default)\n- [ ] CORS configured correctly\n\n### 4. Basic Connectivity\n- [ ] `GET /` returns health status\n- [ ] Server responds to requests\n- [ ] CORS headers present in responses\n- [ ] Cookie parser middleware working\n\n## API Testing\n\n### Authentication Endpoints\n\n#### POST /api/signup (Registration)\n```bash\ncurl -X POST http://localhost:8000/api/signup \\\n  -H \"Content-Type: application/json\" \\\n  -d '{\n    \"name\": \"testuser\",\n    \"email\": \"test@example.com\",\n    \"password\": \"TestPassword123!\"\n  }'\n```\n- [ ] Returns 201 status code\n- [ ] Response includes token\n- [ ] Response includes user object\n- [ ] User created in database\n- [ ] Password is hashed (not plain text)\n\n#### POST /api/login (Login)\n```bash\ncurl -X POST http://localhost:8000/api/login \\\n  -H \"Content-Type: application/json\" \\\n  -d '{\n    \"name\": \"testuser\",\n    \"password\": \"TestPassword123!\"\n  }'\n```\n- [ ] Returns 200 status code\n- [ ] Response includes valid JWT token\n- [ ] Token is usable for protected routes\n- [ ] Invalid password returns 401\n- [ ] Non-existent user returns 404\n\n#### GET /auth/google (Start OAuth)\n```bash\n# Open in browser: http://localhost:8000/auth/google\n```\n- [ ] Redirects to Google login page\n- [ ] OAuth state cookie is set (check cookies)\n- [ ] Code verifier cookie is set (check cookies)\n- [ ] No errors in console\n\n#### GET /auth/google/callback (OAuth Callback)\n```\n# Automatically handled after Google login\n```\n- [ ] Redirects to frontend with token\n- [ ] User created if new\n- [ ] User logged in if existing\n- [ ] Token is valid JWT\n- [ ] Error handled if state doesn't match\n\n### Protected Routes\n\n#### GET /api/myposts\n```bash\ncurl -X GET http://localhost:8000/api/myposts \\\n  -H \"Authorization: Bearer YOUR_TOKEN_HERE\"\n```\n- [ ] Returns 200 with posts array\n- [ ] Returns 401 without token\n- [ ] Returns 401 with invalid token\n- [ ] Only returns user's own posts\n\n#### POST /api/user/change-password\n```bash\ncurl -X POST http://localhost:8000/api/user/change-password \\\n  -H \"Content-Type: application/json\" \\\n  -H \"Authorization: Bearer YOUR_TOKEN_HERE\" \\\n  -d '{\n    \"currentPassword\": \"OldPassword123!\",\n    \"newPassword\": \"NewPassword123!\"\n  }'\n```\n- [ ] Returns 200 on success\n- [ ] Old password validated\n- [ ] New password hashed in database\n- [ ] Token still valid after change\n\n#### DELETE /api/user (Delete Account)\n```bash\ncurl -X DELETE http://localhost:8000/api/user \\\n  -H \"Authorization: Bearer YOUR_TOKEN_HERE\"\n```\n- [ ] Returns 200 on success\n- [ ] User removed from database\n- [ ] All user's data deleted (cascade)\n- [ ] Cannot login after deletion\n\n## Blog Post Operations\n\n#### POST /api/createpost\n```bash\ncurl -X POST http://localhost:8000/api/createpost \\\n  -H \"Content-Type: application/json\" \\\n  -H \"Authorization: Bearer YOUR_TOKEN_HERE\" \\\n  -d '{\n    \"title\": \"My First Post\",\n    \"content\": \"This is my first blog post!\",\n    \"topic\": \"technology\"\n  }'\n```\n- [ ] Returns 201 with created post\n- [ ] Post saved in database\n- [ ] Author name correctly set\n- [ ] Timestamps created\n\n#### GET /api/readblogs\n```bash\ncurl -X GET http://localhost:8000/api/readblogs\n```\n- [ ] Returns 200 with posts array\n- [ ] Posts include like counts\n- [ ] Posts randomized (not sorted)\n- [ ] Limit working (returns max 10)\n\n#### GET /api/readblogs/trending\n```bash\ncurl -X GET http://localhost:8000/api/readblogs/trending\n```\n- [ ] Returns 200 with posts\n- [ ] Posts sorted by likes (highest first)\n- [ ] Maximum 6 posts returned\n\n#### POST /api/readblogs/:id/like\n```bash\ncurl -X POST http://localhost:8000/api/readblogs/1/like \\\n  -H \"Authorization: Bearer YOUR_TOKEN_HERE\"\n```\n- [ ] Returns 200 on success\n- [ ] Like count incremented\n- [ ] Cannot like twice (returns 400)\n- [ ] Preference saved to database\n\n#### POST /api/readblogs/:id/subscribe\n```bash\ncurl -X POST http://localhost:8000/api/readblogs/1/subscribe \\\n  -H \"Authorization: Bearer YOUR_TOKEN_HERE\"\n```\n- [ ] Returns 200 on success\n- [ ] Subscriber count incremented\n- [ ] Cannot subscribe twice (returns 400)\n- [ ] Blogger added to subscriptions\n\n## Error Handling\n\n- [ ] Missing required fields returns 400\n- [ ] Invalid token returns 401\n- [ ] Not found returns 404\n- [ ] Duplicate username returns 409\n- [ ] Server errors return 500\n- [ ] Error messages are descriptive\n- [ ] Error responses are JSON formatted\n\n## Database Integrity\n\n- [ ] All tables exist after startup\n- [ ] Foreign keys enforce referential integrity\n- [ ] Deleting user cascades to posts/preferences/stats\n- [ ] Usernames are unique\n- [ ] Email field exists but not enforced unique\n- [ ] Timestamps auto-populate\n\n## Security Testing\n\n- [ ] Passwords never appear in API responses\n- [ ] SQL injection attempts blocked (parameterized queries)\n- [ ] XSS prevention in place\n- [ ] CORS only allows configured origins\n- [ ] JWT tokens have expiration\n- [ ] httpOnly cookies cannot be accessed by JavaScript\n- [ ] OAuth state validation working\n\n## Performance\n\n- [ ] Responses under 500ms for typical queries\n- [ ] Database indexes present on frequently searched fields\n- [ ] No N+1 query problems\n- [ ] Connection pooling configured\n\n## Logs and Debugging\n\n- [ ] Console logs are informative\n- [ ] Error messages helpful for debugging\n- [ ] No unnecessary verbose logging\n- [ ] Timestamps in logs when relevant\n\n## Production Readiness\n\n- [ ] NODE_ENV set to production\n- [ ] Secure flag enabled on cookies\n- [ ] HTTPS URLs used\n- [ ] Error messages don't expose internals\n- [ ] Sensitive variables in environment\n- [ ] Rate limiting considered\n- [ ] CORS properly restricted\n\n## Frontend Integration\n\n- [ ] Token returned from signup can be used immediately\n- [ ] Token from Google OAuth works for protected routes\n- [ ] CORS allows frontend requests\n- [ ] Error codes match frontend expectations\n- [ ] Response format matches frontend parser\n\n## Deployment Steps\n\n1. [ ] Pull latest code\n2. [ ] Update .env for production\n3. [ ] Run `npm install` to get latest dependencies\n4. [ ] Backup database\n5. [ ] Run migrations if any\n6. [ ] Start server with NODE_ENV=production\n7. [ ] Verify all endpoints responding\n8. [ ] Monitor logs for 30 minutes\n9. [ ] Check frontend can authenticate\n10. [ ] Test Google OAuth flow end-to-end\n\n## Common Issues and Fixes\n\n### Server won't start\n- [ ] Check port 8000 isn't already in use\n- [ ] Verify DATABASE_URL is correct\n- [ ] Check PostgreSQL is running\n- [ ] Review .env file for missing variables\n\n### Cannot connect to database\n- [ ] Verify DATABASE_URL format\n- [ ] Check PostgreSQL credentials\n- [ ] Ensure database exists\n- [ ] Check firewall rules\n\n### Google OAuth not working\n- [ ] Verify GOOGLE_CLIENT_ID is set\n- [ ] Check GOOGLE_CLIENT_SECRET is set\n- [ ] Confirm redirect URI in Google Console matches code\n- [ ] Check BACKEND_URL and FRONTEND_URL\n- [ ] Clear cookies and try again\n\n### Token invalid errors\n- [ ] Check JWT_SECRET matches between requests\n- [ ] Verify token not expired (7 day limit)\n- [ ] Check token format (should have 3 parts separated by dots)\n- [ ] Verify Authorization header format (Bearer TOKEN)\n\n---\n\n**Save this checklist and verify each item before deployment to production.**\n
